Theme Styles Organization

Import custom styles in theme: 
// web/css/source/_theme.less
@import 'components/_header.less';
@import 'components/_footer.less';

.custom-element {
    .lib-css(box-sizing);
    color: @primary__color;
}
Override variables: 
// web/css/source/_variables.less
@primary__color: #ff5501;
@font-family__base: 'Open Sans', sans-serif;

üéØ Variable Overwrite in Theme - Exam Critical!

Magento uses LESS variables for theming. Understanding variable hierarchy is essential.

Variable Fallback Order: 
1. Theme _variables.less (highest priority)
2. Parent theme _variables.less
3. Module lib/web/css/source/_variables.less
4. Magento UI library variables (lowest priority)
Location: <theme_dir>/web/css/source/_variables.less 
// Override Magento default variables
@primary__color: #ff5501;
@primary__color__light: lighten(@primary__color, 20%);
@primary__color__dark: darken(@primary__color, 10%);

// Typography
@font-family__base: 'Roboto', sans-serif;
@font-size__base: 14px;
@line-height__base: 1.6;

// Layout
@layout__max-width: 1280px;
@layout-indent__width: 20px;

// Buttons
@button__background: @primary__color;
@button__border: @primary__color;
@button__color: @color-white;

// Breakpoints (read-only, don't override)
// @screen__xs: 480px
// @screen__s: 640px  
// @screen__m: 768px (mobile/desktop switch)
// @screen__l: 1024px
// @screen__xl: 1440px
Using variables in _extend.less or _theme.less: 
// web/css/source/_extend.less
.my-custom-button {
    background: @button__background;
    color: @button__color;
    padding: @button__padding;
    
    &:hover {
        background: @button__hover__background;
    }
}
‚ö†Ô∏è Important Rules:
  • Create _variables.less BEFORE _extend.less in import order
  • Variables must be defined before use
  • Use @import to inherit parent theme variables first
  • Don't modify vendor files - always override in theme

üéØ Responsive Breakpoints - min vs max

Understanding min-width vs max-width is critical for the exam

Key Breakpoint: @screen__m = 768px

Magento uses 768px as the MAIN mobile/desktop switch point

min-width (Mobile-First) - Magento's preferred approach: 
// Apply styles when screen is ‚â•768px (desktop and up)
.media-width(@extremum, @break) when (@extremum = 'min') and (@break = @screen__m) {
    .header {
        padding: 20px;  // Desktop styles
    }
}

Means: "Apply these styles starting from 768px and above"
Use case: Desktop-specific styles
max-width (Desktop-First): 
// Apply styles when screen is <768px (mobile)
.media-width(@extremum, @break) when (@extremum = 'max') and (@break = @screen__m) {
    .header {
        padding: 10px;  // Mobile styles
    }
}

Means: "Apply these styles up to 768px (767px and below)"
Use case: Mobile-specific overrides
File Structure: 
styles-m.less  ‚Üí Mobile base styles (always loaded)
styles-l.less  ‚Üí Desktop styles (‚â•768px via min-width)

Layout XML:
<head>
    <css src="css/styles-m.css"/>
    <css src="css/styles-l.css" media="screen and (min-width: 768px)"/>
</head>
üî• Exam Critical:
  • min = "from this size and UP" (‚â•)
  • max = "up to this size" (<)
  • Magento uses mobile-first (min-width in styles-l.less)
  • @screen__m = 768px is the key breakpoint

Critical path configuration: 
// view.xml
<vars module="Magento_Theme">
    <var name="critical_css_inline">1</var>
</vars>
  • Inlines critical CSS in <head>
  • Loads remaining CSS asynchronously
  • Improves initial page render time

Adding Inline Styles & CSS from XML

Add CSS/LESS files in layout: 
<page>
    <head>
        <css src="css/custom.css"/>
        <css src="Vendor_Module::css/module.css"/>
        <css src="css/print.css" media="print"/>
    </head>
</page>
Conditional CSS (IE): 
<head>
    <css src="css/ie-fix.css" ie_condition="lt IE 9"/>
</head>
Pass CSS classes via XML arguments: 
<block class="Magento\Framework\View\Element\Template" 
       name="custom.block" 
       template="Vendor_Module::custom.phtml">
    <arguments>
        <argument name="css_class" xsi:type="string">custom-wrapper highlight</argument>
        <argument name="html_id" xsi:type="string">unique-element</argument>
    </arguments>
</block>
Use in template: 
<div class="<?= $block->escapeHtmlAttr($block->getData('css_class')) ?>"
     id="<?= $block->escapeHtmlAttr($block->getData('html_id')) ?>">
    <!-- content -->
</div>
Container with HTML attributes: 
<container name="custom.container" 
           htmlTag="section" 
           htmlClass="content-section bg-gray"
           htmlId="main-content"/>

Dynamic Styles from Backend

Pass configuration to LESS variables: 
// Block class method
public function getCustomColor() {
    return $this->_scopeConfig->getValue('section/group/color');
}
Use in template (inline styles): 
<style>
.custom-element {
    background-color: <?= $block->escapeCss($block->getCustomColor()) ?>;
}
</style>
Or use data attributes + CSS: 
<div class="custom-element" 
     style="background-color: <?= $block->escapeCss($block->getCustomColor()) ?>;">
</div>

‚ö†Ô∏è Always use escapeCss() for inline styles to prevent XSS
